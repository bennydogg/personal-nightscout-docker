# Nightscout Docker Deployment with Cloudflare Tunnel

Docker setup for deploying [Nightscout](https://nightscout.github.io/) CGM with secure external access via Cloudflare Tunnel. Supports multiple instances on a single server.

## Quick Start

### Single Instance

```bash
# Automated setup with Cloudflare tunnel
./setup.sh --domain nightscout.yourdomain.com --setup-tunnel
docker-compose up -d

# Or step-by-step
./setup.sh --domain nightscout.yourdomain.com
./setup-cloudflare.sh --domain nightscout.yourdomain.com
docker-compose up -d
```

### Multiple Instances (one server, multiple users)

```bash
# Create per-user instances with unique ports
./migrate-instance.sh --name alice --domain alice-ns.example.com --port 8081
./migrate-instance.sh --name bob --domain bob-ns.example.com --port 8082

# Check status of all instances
./list-instances.sh
```

Each instance gets its own directory under `/opt/nightscout/<name>/` with independent configuration, database, and containers.

### Migrating an Existing Instance

```bash
# From a running instance (copies data automatically)
./migrate-instance.sh --name alice --domain alice-ns.example.com --port 8081 \
  --source /path/to/old/instance

# From a mongodump file
./migrate-instance.sh --name alice --domain alice-ns.example.com --port 8081 \
  --dump /tmp/backup.gz
```

## Architecture

Each instance runs two containers via docker-compose:

- **nightscout** — `nightscout/cgm-remote-monitor:15.0.6` (internal port 1337, mapped to `HOST_PORT`)
- **mongo** — `mongo:7.0` (configurable via `MONGO_VERSION`) with WiredTiger cache capped at 0.25GB per instance

Port binding is localhost-only (`127.0.0.1:HOST_PORT:1337`). External access goes through a Cloudflare Tunnel. Container names are auto-generated by the compose project (directory name), so multiple instances don't conflict.

### Multi-Instance Cloudflare Tunnel

One tunnel handles all instances via multiple ingress rules in `~/.cloudflared/config.yml`:

```yaml
tunnel: <TUNNEL_ID>
credentials-file: ~/.cloudflared/<TUNNEL_ID>.json

ingress:
  - hostname: alice-ns.example.com
    service: http://localhost:8081
  - hostname: bob-ns.example.com
    service: http://localhost:8082
  - service: http_status:404
```

Register each hostname:
```bash
cloudflared tunnel route dns <tunnel-name> alice-ns.example.com
cloudflared tunnel route dns <tunnel-name> bob-ns.example.com
sudo systemctl restart cloudflared
```

## Configuration

All configuration is via `.env` file per instance (created by `setup.sh` or `migrate-instance.sh`).

### Key Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `HOST_PORT` | Host port for this instance | `8080` |
| `API_SECRET` | API authentication (min 12 chars) | Generated |
| `MONGO_INITDB_ROOT_PASSWORD` | MongoDB root password | Generated |
| `MONGO_CONNECTION` | MongoDB connection string | Auto-configured |
| `MONGO_VERSION` | MongoDB version (4.4, 5.0, 6.0, 7.0) | `7.0` |
| `MONGO_CACHE_SIZE_GB` | WiredTiger cache limit | `0.25` |
| `TZ` | Timezone | `America/New_York` |
| `DISPLAY_UNITS` | `mg/dl` or `mmol/L` | `mg/dl` |
| `AUTH_DEFAULT_ROLES` | Default access level | `readable` |
| `CLOUDFLARE_DOMAIN` | External domain | - |

### Command Line Options

**`./setup.sh`**: `--domain DOMAIN`, `--setup-tunnel`, `--help`

**`./setup-cloudflare.sh`**: `--domain DOMAIN`, `--tunnel-name NAME`, `--non-interactive`, `--help`

**`./migrate-instance.sh`**: `--name NAME`, `--domain DOMAIN`, `--port PORT`, `--source DIR`, `--dump FILE`, `--base-dir DIR`, `--help`

## Management

```bash
# All commands run from the instance directory (e.g., /opt/nightscout/alice/)

# Start/stop
docker-compose up -d
docker-compose down

# Logs
docker-compose logs -f
docker-compose logs -f nightscout
docker-compose logs -f mongo

# Validate
./validate.sh

# Update Nightscout
docker-compose down
docker-compose pull
docker-compose up -d

# List all instances and status
./list-instances.sh

# Cleanup (destroys data for THIS instance only)
./cleanup.sh
```

## Troubleshooting

### Cloudflare Tunnel

```bash
# Debug comprehensively
./debug-tunnel.sh

# Quick fix
./fix-tunnel.sh

# Check service
sudo systemctl status cloudflared
sudo journalctl -u cloudflared -f

# Recreate tunnel
./cleanup-tunnels.sh
./setup-cloudflare.sh --domain your.domain.com
```

### Container Issues

```bash
# Check what's running
docker-compose ps
docker-compose logs nightscout
docker-compose logs mongo

# Test MongoDB
docker-compose exec mongo mongosh --eval "db.adminCommand('ping')"

# Port conflict
# Check what's using a port:
ss -tlnp | grep :8080
# Change HOST_PORT in .env and restart
```

### Instance Issues

```bash
# See all instances and their status
./list-instances.sh

# Validate current instance
./validate.sh
```

## Security

- Port binding is localhost-only (`127.0.0.1`); external access is via Cloudflare Tunnel only
- Containers run with `no-new-privileges` and `cap_drop: ALL`
- `.env` files are created with `600` permissions
- `.dockerignore` prevents secrets from entering build context
- `AUTH_DEFAULT_ROLES=readable` means anyone with the URL can view data; set to `denied` for private access

## Upgrading MongoDB

To upgrade an existing instance from an older MongoDB version:

```bash
# Preview the upgrade plan
./upgrade-mongodb.sh --dry-run

# Upgrade to 7.0 (steps through 5.0, 6.0 automatically)
./upgrade-mongodb.sh

# Or stop at an intermediate version
./upgrade-mongodb.sh --target 5.0
```

The script handles `featureCompatibilityVersion` at each step and backs up data before starting.
